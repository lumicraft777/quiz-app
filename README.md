## 使い方（実行手順）

**重要：`index.html` をダブルクリックして開く（file://）と、Googleの仕様（CORS制限）によりスプレッドシートを読み込めません。必ずローカルサーバー経由で開いてください。**

```bash
# battery-quiz-app フォルダ内で実行（Pythonが入っていればOK）
python -m http.server 8000
```

その後、ブラウザで `http://localhost:8000` を開きます。

1. 起動すると自動でスプレッドシートを読み込み、画面上部の「データ読み込み状況」に結果が表示されます
   - `読み込み完了：◯製品 → 知識問題◯問・実践提案問題◯問を生成しました` → そのまま利用OK
   - `まだデータがありません` → シートにリサーチ結果を貼り付けてから「再読み込み」を押す
   - `読み込みに失敗しました` → 共有設定（リンクを知っている全員が閲覧可）とネット接続を確認
2. モード・カテゴリ・問題数を選んで「クイズを始める」
3. スプレッドシートを更新したら、「再読み込み」ボタンまたはブラウザの更新だけで問題が最新化されます

## スプレッドシートへのデータの貼り付け方

Claude in Chromeのリサーチ結果（タブ区切りの表）を、**1行目が見出し行になるように**シートへそのまま貼り付けるだけです。アプリ側は以下を自動処理します。

- 見出し行は「メーカー名」列を探して自動検出（先頭に多少のメモ行があってもOK）
- メーカー名・製品名が空の行、表の下の「ランキング」などのメモ行、貼り付け時に混ざった2つ目の見出し行は自動スキップ
- セル内のカンマ・改行・引用符にも対応
- 列の**並び順が変わっても見出し名で照合**するので問題なし（見出し名自体は変えないでください）

### 問題化されないデータの扱い（重要）
- 値が「不明」「未確認」「公式未確認」（を含む文字列）・空欄の項目は**その項目の問題を生成しません**
- 「公式未確認」の情報をあえて問題に出したくない場合は、セルの値に「公式未確認」と書いておけば自動的に除外されます
- ダミー選択肢（他製品の異なる値）が3つ揃わない項目も生成されません → **同じ値ばかりの列は問題になりにくい**ので、リサーチ時に値の書き方を製品ごとに揃えすぎない方がクイズ向きです

## 問題の自動生成の仕組み

### 知識問題（[script.js](script.js) パート3）
| 種類 | 例 | カテゴリ |
|---|---|---|
| 値当て | 「◯◯の製品保証年数は？」「蓄電容量は？」「電池材料は？」「負荷タイプは？」「停電時出力は？」「太陽光連携は？」「設置条件は？」「設計寿命は？」「蓄電池タイプは？」 | 保証/容量/電池材料/停電対策/太陽光連携/メーカー比較 |
| 逆引き | メリット・デメリット・特徴・向いている家庭・営業訴求ポイントの文章から製品を当てる | メリット/デメリット・営業トーク・メーカー比較 |
| あり/なし | 「V2H対応しているのは？」「自然災害補償が付帯していないのは？」「AI/HEMS対応は？」 | V2H/保証/メーカー比較 |
| 比較（上級） | 「蓄電容量が最も大きいのは？」「保証が最も長いのは？」（単位付き文字列から数値を自動抽出して比較） | 保証/メーカー比較 |

解説には「主な特徴」「メリット」「営業時の訴求ポイント」のうちシートに値がある要素を必ず含めます。

### 実践提案問題（[script.js](script.js) パート4）
シートの「向いている家庭」「営業時の訴求ポイント」「デメリット」列だけを材料に自動生成します。

1. 各製品の「向いている家庭」等の文章をキーワードスコアリングし、重視カテゴリ（停電対策/電気代削減/初期費用/EV・V2H/保証・安心/設置スペース）を推定
2. **製品選択型**：お客様状況カード（=「向いている家庭」の内容）を提示し、最適な製品を選ぶ。ダミーは重視カテゴリが**異なる**製品のみ（正解が曖昧にならない）
3. **営業トーク判断型（上級）**：同じお客様状況に対し、最も響く訴求トークを4つの製品の「営業時の訴求ポイント」から選ぶ
4. 解説は「なぜ正解か／判断材料になった条件／他の選択肢が弱い理由（各製品のデメリットを根拠に）／営業時の伝え方／注意点」の5要素で構成

**実践問題を増やすコツ**：「向いている家庭」列に、停電・電気代・費用・EV・保証・設置スペースなどお客様の関心が分かる言葉を入れておくと、カテゴリ推定が効いて問題が多く生成されます。

### カテゴリは動的生成
カテゴリの選択肢は「実際に生成された問題」から自動で作られます。データが増えればカテゴリも自動で増えます。

## 将来の拡張：屋根・補助金・エコキュートなどのシート追加

[script.js](script.js) 冒頭の `SHEET_CONFIG` にシートを追加する設計になっています。

```js
const SHEET_CONFIG = [
  { name: "蓄電池メーカー比較", sheet: "", type: "battery" },
  // 例：将来こう増やす（typeごとの問題生成関数の追加が必要）
  // { name: "地域別補助金", sheet: "補助金", type: "subsidy" },
  // { name: "屋根の特徴", sheet: "屋根", type: "roof" }
];
```

新ジャンルの追加手順：
1. スプレッドシートに新しいタブ（シート）を作り、見出し行＋データを入れる
2. `SHEET_CONFIG` にタブ名と新しい `type` を追記
3. その `type` 用の列名マッピングと問題生成関数（`generateKnowledgeQuestions` 相当）を書き、`loadAllData` 内の分岐に追加

シンプルな「用語 → 説明」形式のシートなら、値当て問題と逆引き問題の2種類だけでも十分クイズ化できます。追加したくなったら声をかけてください。
